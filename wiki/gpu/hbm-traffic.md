## TL;DR

GPU 上 GEMM 多是**带宽游戏**：时间 ≈ HBM 流量 / 带宽。分块通过**数据复用**把 HBM 流量从朴素 `2·M·N·K` 压到理想 `M·K+K·N`——复用因子对权重 B = M 倍。这就是打带宽墙的武器。

## 核心概念

### 1. 复用（reuse）
- **取（fetch）** = 从 HBM（显存）把数据搬到快存（寄存器/共享内存）的次数。
- **用（use）** = 数据待在快存里做乘加的计算次数。
- 复用 = 一块数据从 HBM 取 1 次后，在快存里被用多少次。**A_tile 从 HBM 只取 1 次，但在快存里被 N/BN 个 B_tile 复用（用 N/BN 次）**——"和很多 B_tile 算"恰恰是复用本身，不是多取的理由。

### 2. HBM 流量
- 定义：这次计算**实际从显存(HBM)搬到计算单元(快存)的字节数**，类比"从仓库搬了多少货"。
- 为什么关心：带宽固定（如 4090D ≈ 1008 GB/s），**时间 ≈ 流量 / 带宽** → 流量直接决定快慢。
- 朴素流量大根因：抽屉（寄存器/缓存）装不下整个 A/B，每次只拿"当前两个数"，算完即丢，下次重取 → 同一 A[i,k] 对 N 个 j 各取一次（共 N 次）、B[k,j] 对 M 个 i 各取一次（共 M 次）→ 流量 ≈ `2·M·N·K` 元素。

### 3. 两级抽屉模型
- **小抽屉**（寄存器/共享内存，几百 KB）：装 A_tile[BM,BK] + B_tile[BK,BN]，做块内乘加；两者必须同时在此才能相乘。
- **大抽屉**（L2，4090D=72MB）：装更大的 A 行块[BM,K]、B 列块[K,BN]，让它们跨 bj/bi 循环被多个小块复用。
- "取 1 次"跨越哪层循环，取决于数据留在哪级抽屉（HBM→L2 取 1 次、L2→寄存器取 1 次）。
- decode 数字：A[64,2560]bf16 ≈ 320KB 能留 L2；B[2560,11008]bf16 ≈ 56MB < L2 72MB → 容量上 B 能整体留 L2，理论只从 HBM 取 1 次（硬件缓存不保证锁定，但容量可行）。

### 4. 流量公式
- 朴素：`A 每元素取 N 次（M·K·N）` + `B 每元素取 M 次（K·N·M）` = `2·M·N·K`。
- 分块（理想上限）：每块只取 1 次 → `A 切块数×块大小 = M·K`、`B 同理 = K·N` → `M·K+K·N`。
- decode（M=64, N=11008）：权重 B 朴素流量 `M·K·N` vs 分块 `K·N`，比值 = **M = 64 倍**——即"权重朴素搬 64 遍、分块搬 1 遍"的量化来源。

### 5. 装得下才能取 1 次（诚实边界）
- "抽屉装得下"是"取 1 次"的**必要条件**，但还不够：还需遍历顺序配合（`bi,bk,bj` 利于 A 复用、`bi,bj,bk` 利于 B 复用，需二级分块折中）。
- 标准三循环 `bi,bj,bk` 下 A_tile 在 bj 循环被取 `N/BN` 次（除非缓存保留）→ 实际流量 `M·K·(N/BN)+K·N·(M/BM)`，比理想大 `N/BN`、`M/BM` 倍。
- `M·K+K·N` 是**理想上限**，不是分块必然达到；逼近它靠外层 tile 进 L2 + 内层 tile 进寄存器 + 双缓冲隐藏搬移延迟。

## 直觉 / 类比

**仓库(HBM) vs 工作台抽屉(快存)**：
- 朴素：每拧一颗螺丝，都跑去仓库拿一颗螺丝、拿一个螺母，拧完放回，再跑——99% 时间在跑仓库。
- 分块：去仓库一次，搬一大盒螺丝+螺母到抽屉，连拧完一整块产品，抽屉里的料反复用。
- 复用率 = 抽屉里一颗螺丝被用了多少次（分块里 = 输出块另一维大小 N 或 M）。

## 常见误区

- **误区：分块后 HBM 流量一定 = `M·K+K·N`。** 那只是理想上限，要求小块能跨复用循环留在缓存/寄存器；否则退化成 `M·K·(N/BN)+K·N·(M/BM)`。
- **误区："取 1 次"是白送的。** 它需要两级抽屉装得下 + 遍历序配合，否则会退化成取很多次（逼近朴素行为）。
- **误区：朴素写法也有复用。** 朴素三层循环几乎零复用，每个乘加都回 HBM 重取。

## 面试常见问题

- Q：decode GEMM 为什么是带宽游戏？
  A：输出小（M=64 行）、权重大（B 有 56MB），时间主要由把权重从 HBM 搬下来的字节数决定；降低 HBM 流量（提高复用）才能打满带宽。
- Q：分块把流量降了多少？
  A：朴素 `2·M·N·K` → 理想 `M·K+K·N`；对权重 B 而言复用因子 = M（decode 里 = 64 倍）。
