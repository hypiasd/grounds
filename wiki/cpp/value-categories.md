# Value Categories

## TL;DR

左值有名字、有地址、能反复访问；右值是临时的、没名字、用完即弃。最简单的判断方法：**能不能对它取地址**。C++11 进一步细分为 lvalue / prvalue / xvalue 三叉树，xvalue 是左值和右值的交叉地带——有身份但即将被移动。

## 核心概念

### 左值 vs 右值

```cpp
int a = 10;          // a 是左值——有名字有地址，能写 &a
int b = a + 5;       // a + 5 是右值——临时算出来的结果，没有名字

int* p = &a;         // 对左值取地址，没问题
int* q = &(a + 5);   // 对右值取地址，编译报错
```

### 判断规则

1. **能否取地址**：能 → 左值；不能 → 右值
2. **有名字就是左值**：不管原来是什么，进了函数体有了名字就变左值
3. **函数按值返回的结果是右值**：返回值在调用端没有名字

### 值类别三叉树（C++11）

```
           expression
          /          \
      glvalue       rvalue
       /   \         /    \
   lvalue  xvalue  prvalue
```

- **lvalue**（纯左值）：有名字、有地址，能反复找到。例：变量 `a`。
- **prvalue**（纯右值）：临时值、字面量、函数返回值。例：`42`、`a + 1`、`string("hi")`。
- **xvalue**（将亡值）：有名字但即将被移动的对象。`std::move(a)` 的结果就是 xvalue——曾经是 lvalue，被 cast 成 `T&&`，处于"有身份但即将被掏空"的中间态。

xvalue 是左值和右值的交叉地带——它有身份（像左值），但语义上允许被偷（像右值）。

### 关键区分：函数返回值 vs 函数内变量

```cpp
vector<string> build() {
    vector<string> result;
    // ...
    return result;   // result 在这里是左值！它有名字有地址
}
// 调用者拿到的 build() 的返回值才是右值
```

这是最常见的混淆点：**函数的返回值**（在调用者那边）是右值，但**被返回的那个变量**（在函数内部）是左值。编译器看到局部变量被返回、函数即将结束，会自动把它当右值处理（隐式移动或 NRVO），不需要你手写 `std::move`。

## 直觉 / 类比

左值像你家的桌子——有固定位置，你以后还能找到它、操作它。右值像外卖盒——吃完就扔，用完即弃。xvalue 像你打包好准备搬走的箱子——它还在你手上（有身份），但已经贴了"可搬走"的标签（即将被偷）。

## 常见误区

- **以为函数返回的变量是右值**：函数返回值在调用端是右值，但被返回的变量在函数内是左值。编译器自动处理隐式移动，手写 `std::move` 反而阻止 NRVO。
- **以为右值引用类型的变量是右值**：`int&& x = 42;` 中 `x` 是左值——它有名字，能取地址。右值引用是"绑定右值的引用"，但引用本身一旦命名就是左值。详见 [Perfect Forwarding](perfect-forwarding.md)。
- **以为值类别是运行时属性**：左值/右值是编译期的类型标签，不是运行时标记。这也是 `std::move` 能用 `static_cast` 实现的原因——它只改类型标签，不产生机器码。

## 面试常见问题

- **Q: 什么是左值什么是右值？怎么判断？**
  **A**: 最简单的判断方法是能否取地址——能取地址是左值，不能是右值。左值有持久身份（变量、解引用的指针），右值是临时值（字面量、算术表达式结果、函数返回值）。

- **Q: 左值引用和右值引用的区别？**
  **A**: 左值引用 `T&` 绑定到有持久身份的对象，右值引用 `T&&` 绑定到临时对象或将要销毁的对象。右值引用的主要用途是实现移动语义——让容器能"偷"走临时对象的资源而不是拷贝。

- **Q: 一个右值引用类型的变量，本身是左值还是右值？**
  **A**: 是左值。`int&& x = 42;` 中 `x` 有名字、能取地址，所以是左值。语言这样规定是因为有名字意味着可以被多次访问，而"偷完就没"和多次访问是矛盾的。所以编译器默认保守拷贝，需要你显式写 `std::move` 或 `std::forward` 才转回右值。
