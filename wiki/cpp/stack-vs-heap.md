## TL;DR

C++ 对象的出生地点只有两个：栈和堆。栈极快（分配 = 挪栈指针）、自动回收、但大小有限只支持 LIFO；堆任意大小、生命周期灵活、但分配慢且需手动释放。核心决策原则：栈是默认，只在生命周期需超出作用域、大小超限、或需要非 LIFO 顺序时才用堆。

## 核心概念

### 两种分配的本质区别

| | 栈（自动存储期） | 堆（动态存储期） |
|---|---|---|
| **语法** | `int x = 42;` | `int* p = new int(42);` |
| **谁释放** | 编译器，离开作用域自动析构 | 程序员，`delete` 或智能指针 |
| **速度** | 极快（`sub rsp, N` 一条指令） | 慢（malloc/free，涉及空闲链表和碎片整理） |
| **大小限制** | 通常 1-8 MB | 物理内存上限 |
| **生命周期** | 作用域结束即死 | 活到主动释放 |
| **顺序** | 后进先出（LIFO） | 任意顺序 |

### 栈为什么快

分配只需一条 CPU 指令把栈指针往下挪，释放同理——零碎片、零系统调用。堆分配要走 `malloc`，内部要查空闲链表、合并碎片、可能触发 `brk/mmap` 系统调用，开销差几个数量级。

### 什么时候必须用堆

1. **对象大小编译期不确定**（如 `std::vector` 的底层数组、动态大小的缓冲区）
2. **生命周期需要超出当前作用域**（工厂函数返回对象、需要被多个调用者持有）
3. **对象太大，栈放不下**（几 MB 以上的缓冲区——典型栈只有 1-8MB）
4. **需要非 LIFO 顺序的创建/销毁**（栈只支持后进先出，要随机顺序销毁必须上堆）

### 容器本体在栈，数据在堆

```cpp
void foo() {
    std::vector<int> v(1000000);  // v（三个指针）在栈，100 万个 int 在堆
}  // v 析构 → 堆上数据被释放（通过 vector 的析构函数）
```

`v` 本身只是栈上的一个小对象（通常 24 字节，三个指针），它管理的数组在堆上。栈对象析构时，析构函数负责释放它持有的堆资源——这就是 [RAII](raii.md) 的核心。

### 栈上的频繁创建销毁：利用嵌套作用域

```cpp
void process() {
    for (int i = 0; i < 1000; ++i) {
        {  // 每次迭代用 {} 限定作用域
            int buf[256];  // 栈上分配
            work(buf);
        }  // buf 离开作用域，栈空间复用
    }
}
```

栈的 LIFO 特性在嵌套作用域下恰好是优势——离开 `}` 时栈指针回退，下次进入 `{` 时复用同一块空间，零碎片，零分配开销。

## 直觉 / 类比

栈像一摞盘子——只能动最顶上那块，用完抽走，下一块自动露出来。堆像储物间——东西放哪都行，但得自己记位置、自己决定什么时候清掉。

## 常见误区

- **误区一："用 `new` 分配的在堆上，不 `new` 的就在栈上。"** — `std::vector<int> v(100)` 里 `v` 在栈但数据在堆。判断标准不是有没有 `new`，而是对象的内存从哪里来——自动存储期（栈）vs 动态存储期（堆）。
- **误区二："栈对象一定比堆对象快。"** — 快的是分配/释放，不是使用。如果在函数里创建大数组然后 return，栈上的大对象要拷贝（除非 NRVO），堆上可以直接移动指针——使用阶段的拷贝成本可能远超分配成本。
- **误区三："频繁创建销毁小对象就该用堆。"** — 反之。小对象在栈上创建销毁几乎免费，堆上每次都要走 `new/delete`。栈的理想场景恰恰是"频繁创建、用完就扔"——只要生命周期不超出作用域且合计大小不超限。

## 关联

- [Smart Pointers](smart-pointers.md) — 智能指针 = RAII + 堆内存管理，把堆对象的生命周期绑定到栈对象上

## 面试常见问题

- **Q: 堆和栈的区别？**
  **A**: 管理方式：栈由编译器自动分配释放（移动栈指针），堆由程序员手动或通过智能指针管理。速度：栈 O(1) 一条指令，堆需走 malloc/free 的空闲链表查询。大小：栈通常 1-8MB，堆受虚拟内存限制可达 GB 级。生命周期：栈变量随作用域结束自动销毁，堆对象直到显式释放。碎片：栈连续无碎片，堆可能产生碎片。增长方向：栈从高地址向低地址，堆从低地址向高地址。
  *来源：牛客 • Hcoco • 华为OD C++[20260509]；知乎 • linux • [链接](https://zhuanlan.zhihu.com/p/688916716)*

- **Q: 为什么栈的分配比堆快？**
  **A**: 栈分配只需一条 CPU 指令（`sub rsp, N`）把栈指针往下挪——本质是移动一个寄存器。堆分配需要走 malloc，内部要查空闲链表找到合适大小的块、更新链表结构、可能合并碎片、甚至触发 `brk`/`mmap` 系统调用。栈的内存连续且已预热在 cache 中，堆的块分散导致 cache miss 更多。
  *来源：知乎 • 深入浅出cpp • [链接](https://zhuanlan.zhihu.com/p/2023502047459165224)；牛客 • daemon_007 • C/C++ 内存*

- **Q: C++ 程序的内存分区有哪些？**
  **A**: 从低地址到高地址：代码段（存储可执行指令，只读）、数据段（已初始化的全局变量和静态变量）、BSS 段（未初始化的全局/静态变量，程序启动时清零）、堆（动态分配，向高地址增长）、共享区（动态链接库映射）、栈（局部变量/函数参数，向低地址增长）。另外还有常量存储区存放字面量和只读常量。
  *来源：知乎 • linux • [链接](https://zhuanlan.zhihu.com/p/688916716)；牛客 • daemon_007 • C/C++ 内存*

- **Q: 如何判断一个变量在栈上还是堆上？**
  **A**: 函数内直接声明的局部变量（`int a; T obj;`）在栈上——注意容器本身在栈上但其内部数据可能在堆上。通过 `new`/`malloc` 返回的指针指向堆。全局变量和静态变量在静态数据区，既不在栈也不在堆。局部 `static` 变量也在静态数据区——生命周期是整个程序，但作用域局限在函数内。
  *来源：牛客 • Hcoco • 华为OD C++[20260509]*
