# query

## 何时用（触发）
- 用户问："之前学的 X 是怎样的"、"复习一下 Y"、"对比 A 和 B"。
- 用户就仓库里已有笔记的内容提问。
- 区分 query vs. learn：query 回答已有笔记覆盖的内容；learn 处理完全陌生的新概念。

## 目标（完成时仓库应处于的状态）
- 基于仓库笔记给出**带引用**的答案；内容不足时明说并建议 ingest / learn；不凭空编造。
- query 默认不写文件，只有用户要求沉淀时才走 capture。

## 流程
1. **定位**：列 `wiki/` 找相关主题；读对应 `_overview.md` 与具体笔记。
2. **综合策略**（按场景选择）：
   - **单篇笔记足够**：直接基于该笔记作答，引用具体段落。
   - **跨多篇笔记**：先列举相关笔记的要点（各一两句），再对比或串联，最后给出综合结论。
   - **笔记不够但你知道**：明确说"仓库里没有相关笔记，但我可以说说已知的——以下内容未沉淀，建议后续走 learn 存下来"。
   - **完全不知道**：直接说"仓库没有，我也不确定"，建议 ingest 资料或走 learn。
3. **引用粒度**：引用到具体笔记文件（如 `[注意力机制](../llm/attention-mechanism.md)`），如果有多个相关笔记，需逐一标注。
4. **校验（必做）**：确认引用的链接目标确实存在（不要引用不存在的笔记）。
5. **沉淀（可选）**：若答案本身是有价值的新综合视角，提示用户用 `capture` 沉淀；query 本身不强制写文件。

## 边界判断模板

当仓库内容不足以回答时，使用以下标准回复结构：

```
## 仓库内相关内容
- [笔记A](../path/a.md) — 涉及了 X，但没有深入 Y（你的问题）
- [笔记B](../path/b.md) — 提到了 Z，但角度不同

## 仓库外的补充
（如果你知道，可以补充，但必须标注"以下未沉淀"）
- ...

## 建议
- 走 learn 深入学习 Y
- 或 ingest 相关资料补充
```

## 质量示例

**好的 query 回答**：
> 根据 [注意力机制](../llm/attention-mechanism.md) 的笔记，核心思想是让模型在处理每个 token 时动态关注输入序列中不同位置的信息。笔记里用"查字典"做了类比：Query 是你要查的词，Key 是字典的索引，Value 是释义。常见误区是以为 Attention 只在 Transformer 里有——实际上它最早来自 RNN 的 Seq2Seq 模型。
>
> ——有引用、有类比、有误区、定位具体。

**差的 query 回答**：
> 注意力机制是深度学习中的一种技术，用于让模型关注输入的不同部分。它在 NLP 中很常用。
>
> ——没有引用、没有深度、读完和没读一样。

## Gotchas（真实踩过的坑）

- **答案必须可溯源到仓库笔记**，禁止凭空发挥——这是该 skill 存在的意义。如果答案和笔记对不上，用户会产生"笔记白写了"的信任危机。
- **仓库内容不足时明说**，并建议用户 ingest 资料或走 learn，不要硬编。曾有 agent 假装引用笔记但笔记里根本没那个内容。
- **发现过时 / 矛盾 → 提示用户跑 lint**，不要自己默默改笔记。因为你可能在 query 上下文里看不到全貌，静默修改可能引入新矛盾。
- **引用前必须确认链接目标存在**：用户点击一个断链的体验比没有链接更差。
