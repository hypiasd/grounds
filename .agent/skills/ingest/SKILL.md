# ingest

## 何时用（触发）
- 用户发来 PDF / 文章 URL / 仓库地址 / 代码片段。
- 用户说："把这个网页存一下"、"下载这篇论文"、"ingest 这个资料"。

## 目标（完成时仓库应处于的状态）
- `raw/` 有该资料（或一份引用页）；`wiki/<topic>/` 下有摘要页（推荐），通过 `sources:` 指回 raw；互链建立；`_overview.md` 同步；已 `git commit`。

## 流程
1. **落地 raw/**：用户已手动放入 → 直接用；给链接 / 要求下载 → 下载或抓取文本存 `raw/`（文件名小写中划线）；给仓库 → 默认写引用页（URL + 关键说明 + 关注的文件）。
2. **生成摘要页（推荐）**：在 `wiki/<topic>/` 下新建摘要页，遵循 `.agent/conventions.md` 的模板要求（主张式标题、tags、summary 等）。摘要页记录资料讲了什么、关键要点、与学习的关联；`sources:` 指回 raw 原文。
3. **交叉链接 + 更新 `_overview.md`**。
4. **校验（必做）**：若下载了文件，`wc -l` / `ls -l` 确认非空；检查摘要页相对链接可解析；`git status` 确认改动符合预期。
5. **提交**：`git add -A && git commit -m "ingest raw: <资料名>"`。

## 质量示例

**好的摘要页**：
> 论文提出了 XX 方法，核心贡献是解决了 YY 问题。关键洞察是 ZZ。与之前学的 AA 相关（链接），但区别在于 BB。值得深读的部分是 Section 3 的实验设计。

**差的摘要页**：
> 这是一篇关于 XX 的论文。作者是 YY。发表于 2024 年。
> （——只是搬运元数据，没有提炼、没有关联、没有个人判断）

## Gotchas（真实踩过的坑）

- **下载 / 抓取前先确认 URL 与文件名**（小写中划线）；大文件先告知用户体积，因为曾有 200MB PDF 直接下载撑满磁盘。
- **写入后校验非空**：并行写文件曾静默丢失内容（报成功但 0 行）。
- **摘要页不是搬运**：不要复制原文大段文字——写你理解后的提炼，否则笔记失去个人知识库的意义。
