# skill: learn

## 触发
用户提出一个想学的新知识、概念、原理（通常偏向 LLM / 推理框架领域，但不限）。

## 流程

### 1. 老师模式讲解（先对话，不急着写文件）
- 先给**直觉 / 类比**，再给形式化细节。
- 主动指出**常见误区**。
- 不确定处明确说"我不确定"，不要编造。
- 讲完问用户：要不要沉淀成笔记？或继续深入？

### 2. 判断主题
- 先列 `wiki/` 看已有主题，根据内容判断归属。
- 无合适主题 → 新建目录，并创建 `_overview.md`（哪怕先写一句）。

### 3. 沉淀笔记
- 在 `wiki/<topic>/` 下新建 `<note>.md`，套用 conventions.md 的 frontmatter 与正文结构。
- 更新 `wiki/<topic>/_overview.md`（加入新笔记）。

### 4. 收尾
- `git add -A && git commit -m "learn <topic>: <一句话>"`（commit message 即维护日志）

## 注意
- 若用户只是聊天、未要求沉淀，可不写文件，但可在对话末尾提示"需要我沉淀成笔记吗？"。
- 保持和现有笔记的互链，避免孤儿页。
