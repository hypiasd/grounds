# skill: learn

## 何时用（触发）
- 用户问一个新知识 / 概念 / 原理（偏向 LLM / 推理框架，但不限）。
- 用户说："讲讲 X"、"X 是什么"、"帮我理解 Y"、"用老师模式讲一下"。

## 目标（完成时仓库应处于的状态）
- 用户已理解该概念（直觉 + 形式化 + 误区都覆盖）。
- 若用户认可沉淀：仓库新增/更新一篇 `wiki/<topic>/<note>.md`，对应 `_overview.md` 已同步，且已 `git commit`。

## 流程
1. **老师模式讲解**（先对话，不急着写文件）
   - 先给**直觉 / 类比**，再给形式化细节；主动指出**常见误区**。
   - 不确定处明确说"我不确定"，不要编造。
2. **判断主题**：列 `wiki/` 看已有主题，按内容归属；无合适主题 → 新建目录并按 `conventions.md` 模板建 `_overview.md`（骨架先行，内容后续随笔记补充）。
3. **沉淀**（仅当用户认可）：在 `wiki/<topic>/` 下新建 `<note>.md`（套 `conventions.md` 模板）；更新 `_overview.md`。
4. **校验（必做）**：`wc -l <note.md>` 确认非空；`git status` 确认改动符合预期；检查笔记内相对链接目标存在。
5. **提交**：`git add -A && git commit -m "learn <topic>: <一句话>"`。

## Gotchas（真实踩过的坑）
- **不要未经确认就写文件**：先讲，结尾问"要沉淀成笔记吗？"，否则只聊天。
- **写入后必须校验文件非空**：曾发生并行批量写文件时单文件静默落盘失败（报成功但实际 0 行），靠 `wc -l` 才发现。
- **主题名用小写中划线**，且不要预建一堆空目录；主题由内容自然生长。
- **不要重复写 Claude 已懂的通用知识**：只补项目特有约定（见 AGENTS.md / conventions.md）。

## 注意
- 保持和现有笔记的互链，避免孤儿页。
