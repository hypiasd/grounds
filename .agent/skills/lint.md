# skill: lint

## 触发
用户说"体检一下 / lint / 检查仓库"，或定期想维护健康度。

## 检查项

1. **孤儿页**：`wiki/` 下无任何入链、也不被 `_overview.md` 或他处引用的笔记。
2. **缺失链接**：笔记里 `[[...]]` 指向不存在的文件。
3. **矛盾说法**：不同笔记对同一概念描述冲突（尤其 `status: disputed` 相关）。
4. **过时内容**：引用了已被新资料淘汰的说法（对照 raw/ 与 `updated` 日期）。
5. **索引覆盖**：列 `wiki/` 核对每个 `<topic>/` 是否都有 `_overview.md`；笔记是否被 `_overview.md` 或他处引用。
6. **模板合规**：笔记 frontmatter 是否含 `title/topic/created/updated`。

## 流程

### 1. 扫描并产出报告
- 列出问题清单，每条标明：位置、问题类型、建议修复。
- **不要自动修改**（除非用户说"直接修"）。先报告，让用户决定。

### 2. 若用户授权自动修
- 修复后更新 `_overview.md`。
- `git add -A && git commit -m "lint: 修复 <问题摘要>"`

### 3. 收尾（仅报告也记录）
- 报告本身通过对话给出即可；若已修复，`git commit` message 即记录。

## 注意
- Lint 是只读优先：先报告，后执行。
- 冷归档：确认废弃的笔记可移入 `.agent/archive/`，并在原 `_overview.md` 注明。
