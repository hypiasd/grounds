# skill: query

## 何时用（触发）
- 用户问："之前学的 X 是怎样的"、"复习一下 Y"、"对比 A 和 B"。
- 用户就仓库里已有笔记的内容提问。

## 目标（完成时仓库应处于的状态）
- 基于仓库笔记给出**带引用**的答案；内容不足时明说并建议 ingest / learn；不凭空编造。

## 流程
1. **定位**：列 `wiki/` 找相关主题；读对应 `_overview.md` 与具体笔记。
2. **综合作答 + 附引用**：答案链接到具体笔记（`[[...]]`）。
3. **校验（必做）**：确认引用的链接目标确实存在（不要引用不存在的笔记）。
4. **沉淀（可选）**：若答案本身是有价值的新综合视角，提示用户用 `capture` 沉淀；query 本身不强制写文件。

## Gotchas（真实踩过的坑）
- **答案必须可溯源到仓库笔记**，禁止凭空发挥——这是该 skill 存在的意义。
- **仓库内容不足时明说**，并建议用户 ingest 资料或走 learn，不要硬编。
- **发现过时 / 矛盾 → 提示用户跑 lint**，不要自己默默改笔记。

## 注意
- query 默认不写文件；只有用户要求沉淀时才走 capture。
