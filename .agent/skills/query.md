# skill: query

## 触发
用户就已有知识提问、复习、对比、求综合。

## 流程

### 1. 定位
- 先列 `wiki/` 看已有主题，找到相关 `wiki/<topic>/`。
- 读对应 `_overview.md` 与具体笔记。

### 2. 综合作答
- 基于仓库内笔记作答，并**附引用**（链接到具体笔记）。
- 若仓库内容不足，明确说明"仓库里没有，我凭记忆补充，建议 ingest 资料后沉淀"。

### 3. 沉淀（可选）
- 若答案本身有价值（新的综合视角），提示用户是否用 `capture` 沉淀。
- query 本身不强制写文件。

### 4. 收尾
- 通常不改文件；若用户要求沉淀，走 `capture`，并由 capture 负责 commit。

## 注意
- 答案必须可溯源到仓库笔记，避免凭空发挥。
- 发现仓库内容过时 / 矛盾时，提示用户跑 `lint`。
