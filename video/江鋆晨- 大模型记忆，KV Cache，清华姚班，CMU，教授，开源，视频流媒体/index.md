## 从姚班到 CMU：学术成长之路

### 数学竞赛、保送与姚班

江鋆晨的学术起点始于数学竞赛。2006 年，他获得数学竞赛保送资格，2007 年进入清华大学数理基科班——数学与物理的交叉培养项目。然而，在入学后他发现基科班人数众多，恰好此时计算机系姚期智先生创办了一个新班级，入学考试只考数学竞赛内容，他便顺理成章地考入了姚班。

> **[重要]** **姚班当年的独特氛围**
> 姚班在十年前并不是以 AI 为核心——那个年代深度学习远未进入主流视野。当时的姚班探索的是云计算、分布式系统等方向。江鋆晨回忆道，他加入的实验室做的是网络安全，这给他后来的学术选择埋下了伏笔："我特别不喜欢安全，因为安全是禁止你做什么的东西——你得严谨地证明某个东西一定安全，而我更喜欢突破限制。"

在这个早期阶段，他遇到了对他影响深远的导师刘斌老师。"刘斌老师给学生非常多的自由，非常纯粹。你跟他谈研究，他会非常开心——你跟他说的越多，他越开心。"这种氛围让江鋆晨在清华期间建立起对网络系统的浓厚兴趣。

### 选择 CMU 与张辉导师

博士申请时，多数给他发 offer 的地方都是做安全的导师——因为他们看到他的安全背景，以为他会继续走安全路线。但 CMU 是 committee 录取，不指定导师，给了他重新选择的空间。

在 CMU 的开放日，经学长建议，他联系了张辉老师。当时张辉已经在硅谷创业多年（2004 年就开始开公司），正准备离开学术界。但张老师做了一件让江鋆晨至今印象深刻的事：

> **[知识]** **因材施教的导师艺术**
> 张辉老师把他叫到加州，让他在会议室的板前讲了一通自己想做什么。"我当时非常激动，说了我想做这个、想做那个。张老师听了半天，我猜他心里在想着'这是什么东西，做不了'。但他让我讲的过程，是因为他觉得'我只听你讲，才能知道你想做什么，你喜欢什么样的东西'。"后来证明，张老师非常能因材施教——他甚至能让一个学生去做自己本不喜欢的方向，却让学生觉得"这就是我特别喜欢做的事情"。

### Ion Stoica 与"放弃的艺术"

在 CMU 期间，另一位让他深受触动的人物是 Ion Stoica——著名的系统研究者，Spark 的导师之一。江鋆晨对 Stoica 的评价集中在一种罕见的能力：

> **[重要]** **知道何时放弃——Ion Stoica 的学术生涯**
> Stoica 从路由（routing）和 QoS 做到最高潮时说"不做就不做了"，转向 P2P；P2P 在研究最高潮时又说"不做就不做了"，转向大数据（Big Data）；大数据做到最高潮时又"不做就不做了"，开始做 AI Serving。这种能力的关键不在于追逐热门方向，而在于判断"接下去五年什么东西最重要"。
>
> "这种放弃的能力好强大呀。"——主持人月球大叔如此感叹。这与上一期嘉宾分享的"多放弃才能有更多的获得"形成了呼应。

### 本章小结

江鋆晨的学术道路体现了两个关键线索：一是从数学竞赛到姚班到 CMU 的"自由生长"——导师给予的空间比指导更重要；二是对"突破限制"的持续偏好——从抗拒安全研究的"禁止思维"到选择网络系统这个可以自由构建的领域。Ion Stoica 的"放弃的艺术"则给出了一个更高的参照：真正的判断力不在于抓住什么，而在于知道何时放手。

## 视频流媒体的黄金年代

### HTTP 成为流媒体协议的偶然革命

2010 年左右，视频流媒体领域发生了一个类似今天 AI 时刻的拐点。江鋆晨用与当下完全对称的比喻来描述：

> **[重要]** **流媒体的 "Internet Moment"**
> 2010 年的媒体公司面临的情况与今天每个公司要有 AI Strategy 如出一辙：当时的每一个媒体公司都需要一个 Video Streaming Strategy。Netflix、Hulu、HBO、BBC 这些公司突然意识到，自己有视频内容，可以直接通过宽带网络传给用户。但关键问题是：怎么做？
>
> 之前从来没有一套专门为流媒体设计的系统。已有的只有 HTTP——为网页而生的协议。所有网络运营商都支持 HTTP，而 HTTP 在高带宽用户上居然跑得起来。于是所有公司开始野蛮生长，用 HTTP 传输视频。这场自发的革命，与今天大模型领域的爆发有着惊人的同构性。

### 从野蛮生长到系统化研究

正因为工业界在野蛮生长，出现了大量未经系统研究的协议和传输方式。江鋆晨的博士研究方向正是基于这一观察产生：

> **[知识]** **研究机会的诞生**
> "这些所有视频传输的格式和协议，没有研究人员想过：这些协议会对网络有多大的影响？如何用有原则的方法、更好的抽象和算法去优化这一类野蛮生长出来的系统？"张辉老师的公司当时正好在做流媒体系统，先看到了这个需求，但公司里的工程师没有时间做系统化研究——这就是一个典型的学术研究 gap。

Adaptive Bitrate（ABR）算法成为当时的热门研究方向。视频播放器需要根据网络状况动态调整清晰度——这是一个在不确定网络条件下做连续决策的问题。江鋆晨在这个领域发表了多篇论文，积累了系统化研究网络协议的经验。

### 从流媒体到 KV Cache：三维数据的暗线

一个饶有趣味的观察贯穿了整个对话：视频数据和 KV Cache 在本质上有共同之处——都是三维数据。江鋆晨在采访中提到，"同时视频跟 KV Cache 其实都是三维的。"视频数据是时间 $\times$ 空间（宽 $\times$ 高）的三维张量，KV Cache 是层数 $\times$ 序列长度 $\times$ 隐藏维度的三维矩阵。这种对高维数据流的处理经验，为他后来转向大模型基础设施埋下了潜意识的基础。

### 本章小结

十年视频流媒体的研究给了江鋆晨两样东西：一是对"野蛮生长中的系统可以从外部做结构化优化"的信念——HTTP 流媒体最初混乱无序，但研究者可以通过建模和算法使其更可持续；二是对数据流的直觉——视频是时空三维数据，而 KV Cache 是层序维三维数据。这种跨领域的迁移能力，后来在他转向大模型基础设施时发挥了关键作用。

## 存储价格的暴涨与 KV Cache 的浮现

### 为什么存储会在 AI 时代成为问题

对话以一个"劲爆话题"开场：存储价格暴涨。江鋆晨给出了一个反直觉的分析框架——在 AI 时代，所有人的注意力都在算力（Compute）上，没有人考虑存储的能力和容量。但算力越多，对存储的要求也会越多。

> **[重要]** **算力与存储的耦合关系**
> 算力越多 $\rightarrow$ 模型推理中产生的中间数据（KV Cache）越多 $\rightarrow$ 需要存储的数据越多 $\rightarrow$ 存储需求增加 $\rightarrow$ 存储价格上涨（供需失衡）。这个链条中，关键的一环是 KV Cache——模型在计算和运行过程中生成的数据，是目前对存储压力非常大的一个原因。

江鋆晨半开玩笑地说："这有可能是我们是导致价格上涨的原因之一。"这句话背后的实际含义是：LMCache 团队是最早系统性研究 KV Cache 存储问题的团队之一——他们既看到了问题，也在推动解决这个问题。

*存储价格暴涨的背后逻辑：算力增加 $\rightarrow$ KV Cache 增加 $\rightarrow$ 存储需求激增*

*视频画面时间区间：00:00:00--00:08:00。*

### KV Cache 是模型的"记忆"

在对话的早期，江鋆晨就抛出了一个关键定义——KV Cache 是模型的记忆。这不是一个比喻，而是一个技术事实：

> **[重要]** **KV Cache = 模型的记忆**
> 模型在"阅读"一段文本时，会将输入转化为内部表示——Key-Value 矩阵。这个矩阵就是模型对这段文本的"理解"或"记忆"。如果这个记忆被丢弃，下次模型看到同样或相似的文本时，必须从头重新计算。KV Cache 的本质就是把模型的"理解过程"的中间产物保存下来，避免重复劳动。
>
> 这与人阅读一本书的过程类同：你读完一本书后，脑子里有了对书的理解（记忆），下次别人问你这本书的内容时，你不需要重新翻一遍——可以直接从记忆中提取。模型也应该如此。

### 本章小结

存储暴涨不是简单的供需问题，而是 AI 时代结构性变化的一部分。当所有人都盯着算力时，存储作为"看不见的基础设施"正成为瓶颈。KV Cache 既是造成存储压力的原因，也是解决问题的钥匙——因为一旦能被有效存储和管理，它就能成为模型的持久记忆，而非一次性消耗品。

## 在学术与工业之间穿梭

### 写产品代码的那段经历

博士毕业后，江鋆晨有一段在工业界写产品代码的经历。这段经历对他的研究产生了意想不到的影响：

> **[重要]** **工业界的真实问题是最好的 PhD 课题来源**
> "在工业界直接去写产品代码、做工程——那次对我研究的帮助是最大的。"为什么？因为工业界工程师不会从论文里找问题来做——他们面对的是真实的用户痛点、真实的系统瓶颈。而这些真实问题，恰恰是最有研究价值的学术课题。
>
> "工程师们给我的问题，后来变成了我的 PhD 研究课题。"这种"从工业界获取问题 $\rightarrow$ 在学术界做系统化研究 $\rightarrow$ 最终回到工业界落地的循环，是他整个职业生涯的核心模式。

### LMCache 的诞生

LMCache 最初的起点是一个学术项目。在 ChatGPT 爆发后，江鋆晨和他的学生开始思考一个问题：为什么每次向大模型提问相同的上下文，它都要重新计算一遍？是否存在一种方法，能把模型的计算结果以"记忆"的形式保存下来？

> **[知识]** **从学术项目到开源项目的跃迁**
> LMCache 作为学术项目开始后，团队越来越清楚地意识到：要做的事情太大了——要给整个工业界做一套新的基础设施，还要把研究 idea 源源不断输送到工业界——这绝对不是学术界或一个开源项目能单独完成的事情。"必须得拉投资，必须得开公司，必须得让工业界一起参与进来。"于是，学术项目自然演化成了创业公司 TensorMesh。

江鋆晨的职业生涯有一个稳定的模式：从工业界的真实问题出发，在学术界做系统化、原理性的研究，再将研究成果以开源和创业的形式推回工业界。这种"工业-学术-工业"的闭环，使他的研究始终扎根于真实需求，而非论文写作的自我循环。

## 判断力：高价值问题、科学史与学生

### 如何识别高价值问题

当被问到"如何判断什么问题是值得投入的高价值问题"，江鋆晨的回答出人意料地务实：

> **[重要]** **判断高价值问题的三个原则**
> 1. **不要跟风**：当所有人都涌入一个方向时，那里大概率已经没有高价值问题留给新进入者了。
> 2. **寻找结构性缝隙**：在工业界"已经在做但还没做好"和学术界"还没人研究"之间，存在大量高价值问题。这些问题工业界没时间系统化，学术界还没发现。
> 3. **看底层规律而非表面热度**：视频流媒体的崛起不是因为某家公司做了什么，而是因为宽带普及这个底层变量——AI 时代同理。

### 多读科学史

江鋆晨反复强调科学史的重要性，这在外人看来似乎与做技术创业无关，但他有自己的理由：

> **[知识]** **为什么技术人要读科学史**
> 科学史让你看到技术演化的底层规律。比如：
>
> - HTTP 成为流媒体协议不是因为它"最好"，而是因为它是唯一所有人都支持的协议——这与今天 OpenAI API 成为事实标准的逻辑一脉相承
> - 每一次技术浪潮都有"被遗忘的方向"——它们不是错了，而是早了（timing matters more than correctness）
> - 真实的技术演进从来不是线性的——读历史可以培养对"非线性"的直觉

### 如何找学生

作为芝加哥大学的教授，江鋆晨的招人标准同样体现了他的价值观。他不看重简历上的"亮点词"，而是看重：

- 是否真正有热情——不是"应该做 AI"而是"忍不住要做 AI"
- 是否能独立思考——你的想法来自哪里，是不是你自己的
- 是否有好奇心——你会不会因为一个问题有趣而去研究它，不管它当下火不火

### 本章小结

判断力不是某种玄学能力，而是可以被训练的思维方式：多读历史知道什么规律在重演，多接触工业界知道什么问题是真实的，多看底层变量知道什么变化是结构性的。找学生也是如此——与其找"符合标准"的人，不如找有内在驱动力的人。

## All in ChatGPT：范式的跳转

### 从流媒体到大模型：一个人生转折

2012 年 AlexNet 出现时，深度学习只是计算机视觉领域的一个小众方向。江鋆晨回忆道，那时清华孙茂松老师的一个博士生出去开会回来，像讲新闻一样告诉大家："深度学习这个东西以后是未来，我们得做。"而当时大多数人还在用 SVM 和 Linear Regression 入门机器学习，"大家都不知道深度学习是什么"。

真正让他做出人生中最重大转向的，是 ChatGPT 的出现。这个瞬间，他看到了一个与视频流媒体完全对称的历史机遇：

> **[重要]** **ChatGPT 作为 "Internet Moment"**
> ChatGPT 之于大模型时代，就如同 2010 年 HTTP 视频流媒体的爆发之于网络时代。两个时刻共享同样的特征：
>
> - 一个新技术突然证明自己能用（且足够好）
> - 所有公司立刻意识到"我们必须有这个东西"
> - 但没有人知道怎么正确地、可持续地做
> - 研究者有机会在野蛮生长中建立系统化的方法论

正是基于这个判断，他决定放弃做了十年的视频流媒体领域，全身心投入大模型基础设施。"这不是跟风，而是重复已经验证过一次的模式。"

### 为什么选择做基础设施而非模型

很多人会问：既然看到了 ChatGPT 的浪潮，为什么不自己做模型？江鋆晨的回答反映了典型的系统研究者思维：

> **[知识]** **做模型 vs 做基础设施**
> 做模型的公司已经有很多了，而且竞争极其激烈。但所有模型都需要运行，运行就需要基础设施。KV Cache 是所有 Transformer 模型都共有的组件——不管你是谁家的模型，都要面临同样的问题。这就像在视频流媒体时代，不去做 Netflix 那样的内容公司，而是去研究所有视频公司都需要的传输协议。这是一个更宽、更深、也更持久的机会。

### 本章小结

"All in ChatGPT"不是一个投机决策，而是一个模式识别的结果。十年前他在视频流媒体爆发时进入，看到了"野蛮生长中的系统化研究机会"；十年后他在大模型爆发时看到了完全相同的机会结构。这种模式识别能力，比任何具体的技术知识都更为稀缺。

## 深入理解 KV Cache

### KV Cache 是什么

这是视频中最技术的部分。江鋆晨用一个直观的类比来解释 KV Cache 的工作原理：

> **[重要]** **KV Cache 的直觉理解——程序员读代码**
> 你是一个非常聪明的程序员，我给了你一个程序让你去分析，告诉你说"能不能在这个程序里面加一个新的功能"。
>
> 你做的第一件事情是肯定得先读这个程序——模型也是一样。人会把程序理解成脑子里能记住的结构和记忆，模型也有这样的记忆——这个记忆就是 KV Cache。它是一大坨浮点数，是一个三维矩阵，但这是模型"记住"新知识的方式。
>
> 下一次如果我再让你在这个程序上加另一个功能，如果你把之前读程序的记忆给忘了（现在很多模型确实在干这件事），你就得从头再读一遍——那 90% 的输入都是同一个程序，为什么要重复计算呢？

### Prefill vs Decode：算力分配的"反直觉"真相

江鋆晨提出了一个非常重要的观察——绝大多数人对模型推理的算力分配存在误解：

> **[注意]** **算力分配的反直觉真相**
> **人的感官**：你盯着屏幕等模型输出，感觉 90% 的时间都在等它生成（Decode）。
>
> **实际算力**：90% 的算力花在了输入处理（Prefill）上。
>
> **为什么**：在 Agent 场景中，输入往往是几万、几十万甚至几百万 Token（上下文 + 工具调用结果 + 历史记录），而输出通常只有几千 Token。"你现在的输出永远都会变成以后的输入"——这句话是对这个现象最精辟的概括。
>
> **含义**：硬件公司优化 Decode 速度（因为用户能感知到），但真正的算力瓶颈在 Prefill——这正是 KV Cache 发力的地方。如果你能复用之前 Prefill 的计算结果（即 KV Cache），就能跳过 90% 的重复计算。

### LMCache 的设计哲学

LMCache 要做的事情，可以概括为一个词：解耦。

> **[重要]** **LMCache 的核心设计哲学——强解耦的 KV Cache Layer**
> 在现有的推理架构中，KV Cache 通常被强绑定到具体的推理引擎上（比如 vLLM 的 PagedAttention 管理的内存）。这种绑定有几个问题：
>
> - 你想换推理引擎，KV Cache 的格式和管理就得跟着变
> - 你想把 KV Cache 存到不同的存储介质（DRAM、SSD、远程存储），没有统一接口
> - 你想在不同模型之间共享或迁移 KV Cache，几乎不可能
>
> LMCache 的答案：做一个独立的、强解耦的 KV Cache Layer，让它处于推理引擎和存储系统之间。不管用什么引擎（vLLM、SGLang、TensorRT-LLM）、什么存储（本地 DRAM、NVMe、远程对象存储），LMCache 提供统一的接口来管理和传输 KV Cache。

### 为什么开源是必须的

江鋆晨在采访中坦诚地讨论了开源的风险。做开源项目的公司很多，但真正成功的很少——Spark 是直接从工业界需求中生长出来，硬生生干掉了当时的闭源方案。LMCache 也走了这条路：

> **[知识]** **开源的哲学**
> "开源这件事情就是让大家有选择。最大的需求就是：我用你的东西，也可以用别人的东西，我随时可以在各个层上换选择。"做解耦的 KV Cache Layer，开源是天然的选择——因为它要成为所有引擎和所有存储之间的"中间人"，闭源会破坏这个定位。
>
> "如果说某一家公司更愿意接入你们这个标准，那么可能的话，其他的模型厂商就更愿意用这一家的存储介质或推理引擎。"这种网络效应，正是 LMCache 试图通过开源建立的标准地位。

### 章末小结

KV Cache 不仅仅是一个技术优化手段，它代表了一种看待模型计算的新范式：将模型的计算结果视为一种可复用、可管理、可优化的资产（asset），而非一次性消耗品。从 Prefill 到 Decode 的算力分配真相，到 LMCache 的强解耦架构，江鋆晨的核心论点是：KV Cache 是模型推理的"记忆层"，而这个记忆层需要独立的基础设施来管理。

## Agent 与模型原生记忆

### Agent 为什么需要更好的记忆管理

Agent 的本质是"模型与环境的多次交互"。与单次对话不同，Agent 在完成一个任务的过程中，需要和外部环境（工具、数据库、其他 Agent）反复交互：

> **[重要]** **Agent 场景对记忆管理的挑战**
> - 交互次数增多 $\rightarrow$ 需要记住的上下文激增
> - 记忆不是简单的堆积——需要不停地 shuffle、取出最相关的部分
> - 越复杂、越长的任务，记忆管理的方式也需要越复杂
> - 传统方案用"人话"管理记忆（存文本、Code、图片），但模型需要的是"模型可以理解的记忆"

### 模型原生记忆 vs 文本记忆

江鋆晨提出了一个关键区分——当前绝大多数记忆管理系统以"人能理解的形式"管理记忆（存储文本、对话历史等），然后在每次调用时把这些文本重新输入给模型。但 LMCache 管的不是这些文本，而是"文本在人话层面的镜像"——即文本经过模型处理后的 Key-Value 表示：

> **[重要]** **模型原生记忆的优势**
> - **直接可用**：KV Cache 的格式是模型神经网络的内部表示，模型可以直接"读取"，不需要重新编码
> - **保留注意力信息**：KV Cache 中不仅包含内容，还包含模型对不同 Token 的注意力权重——哪些部分更重要、哪些部分可以忽略——这些信息在文本形式中会丢失
> - **可分析**：通过分析 KV Cache，可以理解模型到底记住了什么、对哪些 Token 注意力更集中
> - **可优化**：可以从 KV Cache 中提取信号来优化模型未来的行为，而不仅仅是被动存储

### 从"记下来"到"理解记忆"

江鋆晨描绘了一个更远大的愿景——KV Cache 不只是拿来省的，还能拿来"理解模型"：

> **[知识]** **KV Cache 的长期愿景**
> "你存下来的是记忆，除了复用，你还能做什么别的？你可以去理解一下它记忆里面是什么东西——模型到底记住了什么，模型到底对哪些 Token 注意力更集中，哪些 Token 它觉得更重要。甚至你还可以去优化它的记忆，让它以后的行为可以更好。"
>
> 这意味着 LMCache 的未来定位不只是"KV Cache 存储公司"，而是一个"帮助大家理解大模型本身工作原理"的公司——从存储到洞察，这是一条从工程到科学的升维路径。

### 章末小结

Agent 时代的到来使得记忆管理从一个"优化问题"升级为"架构问题"。当模型需要在长时间、多轮交互中保持一致的认知状态时，仅靠外部文本记忆是不够的——模型需要的是原生格式的、保留注意力信息的记忆。KV Cache 的复用和优化，构成了这一新型记忆管理的技术基础。

## 未来基础设施与职业建议

### OpenAI API = 新时代的 IP 层

江鋆晨在社交媒体上反复讨论的一个观点是：OpenAI API 是新时代的 IPv4。他特意强调"这跟我们公司战略没有直接关系"，但作为一个资深的网络系统研究者，这个类比揭示了对 AI 基础设施未来的深层思考：

> **[重要]** **从 IP 层到 API 层——基础设施的"细腰"结构**
> 在互联网架构中，IP 层（特别是 IPv4）是所有应用都承认的通信接口——位于 TCP 和物理层之间的"细腰"。IP 层上做创新非常困难（IPv6 至今未能完全取代 IPv4，因为所有人都已经在用 IPv4）。
>
> 在 AI 时代，OpenAI API 的接口格式（API compatible）正在扮演相同的角色。所有模型提供商都在兼容 OpenAI 的 API 格式——就像所有网络应用都兼容 IP 协议。一旦这个格式稳定下来，它将成为 AI 生态系统中不可撼动的"细腰"：
>
> - **在 API 层之上**：应用可以百花齐放，用任何模型的 API
> - **在 API 层之下**：每个模型厂商可以在自己的技术栈上创新
> - **API 层本身**：一旦稳定，很难被取代——即使有更好的替代方案（就像 IPv6 之于 IPv4）

## 总结与延伸

### 讲者的核心脉络

江鋆晨在整个对话中展示了一条清晰的叙事线：

1. **学术选择**：从数学竞赛到姚班到 CMU——每一次选择都是"去找更大的空间"
2. **研究模式**：从工业界获取真实问题 $\rightarrow$ 在学术界做系统化研究 $\rightarrow$ 以开源和创业推动落地
3. **范式跳转**：从视频流媒体到 KV Cache——看到了完全相同的"Internet Moment"结构
4. **核心信念**：KV Cache 不仅是存储优化工具，更是模型的"记忆层"——一种可管理、可分析、可优化的新资产类型
5. **创业实践**：在无人竞争的领域先发，用信念推动共识，用信任构建团队

### 结构化的核心洞察

从这场对话中可以提炼出几个跨领域的核心命题：

> **[重要]** **KV Cache 的三大命题**
> **命题一（工程命题）**：模型推理中 90% 的算力投入在 Prefill（输入处理），而大多数硬件优化集中在 Decode（输出生成）。KV Cache 通过复用 Prefill 的中间结果，直接命中算力分配的结构性矛盾。
>
> **命题二（架构命题）**：KV Cache 需要成为一个强解耦的独立层（Layer），从推理引擎和存储介质之间分离出来，在开源生态中形成标准。
>
> **命题三（愿景命题）**：KV Cache 的长期价值不仅在于"省钱"——它是理解模型行为、优化模型表现的关键数据资产。模型原生记忆的管理，将成为 Agent 时代的基础设施。

### 职业判断力的底层逻辑

> **[重要]** **判断力的三个层次**
> **第一层——识别模式**：看历史以理解规律（读科学史），看底层变量以理解结构（宽带普及 $\rightarrow$ 流媒体爆发，计算成本下降 $\rightarrow$ LLM 爆发）
>
> **第二层——放弃的勇气**：像 Ion Stoica 那样在最高潮时离开一个领域。最难的不是选择做什么，而是选择不做什么。
>
> **第三层——推动共识**：在某个方向被工业界广泛承认之前，用自己的信念和执行力去推动它——这就是"现实扭曲力场"的正向含义。
