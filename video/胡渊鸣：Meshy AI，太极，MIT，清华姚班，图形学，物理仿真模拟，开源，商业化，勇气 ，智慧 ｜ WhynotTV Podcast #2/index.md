## 引言：WhynotTV 与胡渊鸣的对话

WhynotTV Podcast 主持人泰然与 Meshy AI CEO 胡渊鸣进行了一场长达三个小时的深度对话。胡渊鸣小学便开始编写物理仿真程序与游戏开发，本科就读于清华姚班，后赴 MIT 攻读博士。他是世界上最优秀的计算机图形学专家之一，博士期间独立全栈开发出开源并行计算框架「太极」（Taichi），突破单 GPU 十亿粒子的仿真极限。2021 年博士毕业后创立太极图形，经历 18 个月商业化探索后转型推出 3D 生成式 AI 平台 Meshy，现服务超 400 万用户，年营收增长超十倍。

## 童年编程启蒙与姚班岁月

### 小学：VB6 与物理仿真的萌芽

胡渊鸣的编程生涯开始得非常早。「很小的时候，我就知道自己很喜欢写程序。一天不写程序，我浑身就难受。」小学时，他从天空软件园下载了 6MB 的 Visual Basic 6.0 安装包，开始编写最基础的物理仿真程序。

他回忆了一个特别有启示性的早期经历：用上下左右键操控屏幕上的物体移动时，他发现「只让物体朝前移动的话，会非常不真实」。于是他自发地做了一个设定——用上下左右操控的应该是**加速度**，而不是**位移**。这在本质上是牛顿第二定律 $F=ma$ 的朴素直觉。一个小学生，在没有正式物理课的情况下，自己摸到了经典力学的门槛。

他还用 VB6 实现了**质点弹簧系统**（Mass-Spring System）来搭建虚拟建筑物，类似现实中的磁力棒玩具：一堆球和弹簧连接在一起。

### 高中：刚体仿真与多米诺骨牌

进入高中后，胡渊鸣开始研究**刚体动力学**（Rigid Body Simulation）。他在电脑上模拟多米诺骨牌——因为在现实中堆多米诺骨牌「没有场地，一不小心碰一下就全部完蛋了，我觉得在电脑里面做可能更有意思」。

正是在这个过程中，他第一次意识到了**建模的不精确性**：「两个刚体之间的 collision，或者它们的摩擦 friction，这些的 model 很多都是一些近似 model。」这个问题后来成为他博士研究的核心关切。

> **[知识]** **物理直觉 vs 形式教育**
> 胡渊鸣的案例说明，深刻的物理直觉可以完全独立于课堂教育而生长。他将自己的启蒙归因于「想创造世界」的内在驱动力——写代码是他表达这种驱动力的媒介，而非目的本身。当他发现模拟不够「真实」时，好奇心自然驱使他去追问和修正模型。

### 清华姚班：竞争、挫败与自我认知

凭借竞赛成绩，胡渊鸣进入清华姚班。他以「收获非常非常大」形容这四年，但收获的来源并非课程本身，而是**一帮非常厉害的同学**。毕业近十年后，姚班同学依然每年聚会，这种 peer effect 的影响远超任何单一课程。

他回忆起大学第一门课「计算机应用数学」的挫败经历：「这门课和一般的学院课不太一样，一般你到清华会先学微积分、线性代数、概率论……那门课好像是直接就 assume 你会了这些东西。」最离谱的是，老师在考后说：「这个知识点我上课没来得及讲，所以呢，我就把它出成了考试题目。」

本科期间，胡渊鸣做过 AI 研究和图形学研究。一个值得注意的事实是：「最后能发出论文的全部都是 AI 的研究，graphics 一篇都没发出来。」这与他后来在 MIT 坚定选择做图形学形成鲜明对比——他并不是因为早期成功才选择图形学，而是出于对这个问题本身的热爱。

> **[重要]** **「我只能做自己 enjoy 的事情」**
> 在 MIT 博士第一年的痛苦中，胡渊鸣学会了一件至关重要的事：「我只能做自己 enjoy 的事情。如果说一个事情我没有那么 enjoy，I would rather not do it——我宁愿不做它，我也不会去开始一段痛苦的经历。」
>
> 这个认知在创业后变得更加锋利：当你需要 7×24 小时投入一件事时，如果不 enjoy，你是撑不下去的。

### 本章小结
从 VB6 到姚班，胡渊鸣的早期经历展示了一个关键模式：持续的内在驱动力 {} 外部的成功定义。他并非按「最优路径」规划人生——他选择了更难、更不确定但自己更 enjoy 的图形学方向。这种 self-awareness（自我认知的清晰度）是他后来所有重大选择的基础。

## MIT 博士：太极的诞生与自我颠覆

### 第一年的黑暗与幸存

胡渊鸣到 MIT 后遭遇了严重的适应危机。导师的研究兴趣与他并不一致，大量 founding project 的压力让他一度想过退学。但他挺了过来，原因有二：一是当时「还算比较 productive」，发了不少论文；二是他与几位清华学长聊了聊，获得了精神支持。

他将自己的幸存归因于运气：「你就是 genetically 抽中了一个比较好的基因，所以你恰好在这个时代，又需要你这个基因能做的事情。所以我觉得可能 99% 都是运气。」他甚至认为「你能努力也是运气的一部分，因为你有这个运气，所以你知道你喜欢什么，所以你可以努力。」

> **[注意]** **高成就者的「运气认知」陷阱**
> 胡渊鸣的「99% 都是运气」不是一个谦辞，而是他反复思考后的真实世界观。它有两个面向：正面是保持谦逊、持续学习；负面是可能低估自己方法论的价值。对于普通读者而言，重要的不是争论这个比例，而是理解：他对「知道自己喜欢什么」这件事的珍视远超过对智力的珍视。

### 太极：从造轮子到造生态

转机出现在博士中后期。胡渊鸣找到了一个独一无二的角度：**从底层编译器开始，自造一套用于物理仿真的并行编程语言**。这个项目的灵感来自他对 CUDA 的观察：CUDA 让每一个 C/C++ 程序员都可以写 GPU 并行程序，但 C++ 正逐渐被 Python 取代。那么，能不能让 Python 程序员也能写出高性能的并行仿真代码？

太极（Taichi）应运而生。最让他自豪的是，太极**仅用两行代码**就能把一个手写的 CUDA 仿真程序变成可微的（differentiable）：

```
import taichi as ti

ti.init(arch=ti.gpu)

# 原有的物理仿真代码
@ti.kernel
def simulate():
    # ... 几百行 CUDA 级仿真逻辑 ...

# 两行代码实现自动微分
simulate.grad()  # 自动生成反向传播的计算图
loss = compute_loss()
loss.backward()  # 梯度反向传播
```

胡渊鸣解释这种兴奋：「我觉得特别有意思，我也觉得只有自己造了一个编译器，才能做到这个事情。」

### Full-Stack 的独狼模式

太极是一个 solo project——胡渊鸣一人把整个 full stack 都做了一遍：

- 从 **PTX 汇编**一路写到知乎文章宣传
- 扮演**产品经理**：分析 simulation 用户的真实需求（他自己就是目标用户）
- 扮演**科学家**：了解最新技术如何 convert 成好产品
- 扮演**工程师**：C++ 性能优化、多线程并行编译、编译 cache、中间表示设计
- 扮演**CEO**：把以上成果整合起来，让更多人能用上

他将这种状态形容为「某一种叠加态，或者说一种雌雄同体的状态」——自己闭环把整个事情做出来。

> **[重要]** **太极的核心技术洞察**
> **编译器即平台**：通过 Python 前端 + 自研编译器后端的架构，太极让用户用 Python 编写仿真代码，编译器将其编译为可在 GPU 上高效运行的并行程序。这使得物理仿真从「需要精通 CUDA」变成了「会 Python 就能做」，大幅降低了高性能仿真的门槛。太极是物理仿真领域的「PyTorch 时刻」。

### 高目标驱动：求上得中

胡渊鸣在博士阶段确立了一个看似不可能的目标：「一年 PhD 毕业」。他明知这不现实，但认为**「求上而得中，求中得下，求下而不得」**——设立高目标本身就是一种动力机制。

实际上，他第三年就发现「不知道在读博士的话还能做些什么了」——paper 已经有足够影响力，导师也让他提前毕业。他并非因为完不成才降低标准，而是发现**自己在学术界的边际贡献已经趋于零**。这直接促成了后来的创业决定。

### 本章小结
MIT 博士阶段是胡渊鸣人生中第一次重大的「自我颠覆」：从跟随导师做 founding project 的博士生，蜕变为独立造出太极编译器的 creator。太极不仅是一项技术成就，更是一次 full-stack 能力验证——证明他可以从汇编写到产品、从代码写到社区。这种闭环能力后来成为他创业的核心底气。

## 图形学的演变与瓶颈

### 从渲染到仿真：三代演化

胡渊鸣将计算机图形学的发展分为三个阶段：

- **第一代（1995 年以前）**：以可视化和真实感渲染（photorealistic rendering）为核心。这一时期的 graphics 本质上是「把数学变成好看的图」。
- **第二代（1995--2015 年）**：物理仿真的崛起。以 Ron Fedkiw 等人的烟雾模拟（smoke simulation）为标志性起点，graphics 开始关注如何用数值方法复现物理世界的动态过程。流体、刚体、弹性体仿真相继成熟。
- **第三代（2015 年至今）**：AI 的渗透与融合。以 differentiable rendering、neural radiance fields（NeRF）、3D Gaussian Splatting 等技术为代表，graphics 和 machine learning 的边界逐渐模糊。

胡渊鸣指出，graphics 领域为 AI 时代奠定了硬件基础：**「AI 用的硬件也是基于 graphics 最早的时候演变出来的。」** GPU 本身就是为了图形渲染而设计的，而今天 GPU 已成为所有深度学习的基石。

### 渲染研究的瓶颈

在博士后期，胡渊鸣对 graphics 学术界的创新能力产生了怀疑。「大概是在第三年吧，就是快毕业的时候，那个时候大家搞 rendering 已经变成一些已有技术的排列组合了。」

他举了一个例子：有一篇论文叫 MCMC PPM（自适应马尔科夫链蒙特卡洛统一路径采样与顶点连接与合并），「就是把 MCMC 和 unified path sampling 和 adaptivity 给整到了一起。当然还是不错的、有意思的工作，但这样的模式——把已有技术进行排列组合——说明这个领域的 core innovation 空间已经在收窄。」

> **[注意]** **学术创新的边际递减**
> 胡渊鸣的观察指向一个普遍现象：当一个工程/科学子领域进入成熟期后，新发表的论文往往是对已有技术的「排列组合」而非真正的新范式。这在 rendering 领域尤为明显——路径追踪、光子映射、重要性采样等核心思想在 20 年前就已奠定，后续工作更多是工程优化和边缘改进。

### 为什么选择物理仿真而非渲染

胡渊鸣选择物理仿真而非传统渲染作为研究方向，背后有一个深刻的技术判断：**仿真的问题比渲染更难，但也更有结构性机会**。渲染的核心任务是「给定场景描述，产出一张图」——这本质上是一个从 3D 到 2D 的投影问题。而仿真的核心任务是「给定初始状态和物理定律，预测系统的动态演化」——这是一个更高维的、涉及时间序列的预测问题。

更重要的是，仿真天然与 AI 互补：**「牛顿第二定律是准的，相对论也是准的，但很多 boundary condition 可能是不准的。」** 这意味着仿真中最难的部分并非核心物理定律（已经知道），而是那些「在真空环境下 work 的物理定律，在现实有空气阻力、物体表面不是刚体、边界条件和你想的完全不一样」的情况下如何修正——这正是 machine learning 擅长填补的 gap。

### 本章小结
图形学经历了从渲染到仿真再到 AI 融合的三代演化。在渲染研究陷入边际递减之时，物理仿真成为一个更有结构性机会的方向——它既保留了第一性原理（物理定律）的可解释性，又为 data-driven 方法（修正边界条件）留下了充分的接入点。

## 物理仿真与 AI 的融合

### 传统 Simulator 的根本问题

胡渊鸣对现有仿真器（Isaac Sim、MuJoCo、Drake、PyBullet、Genesis 等）的批评直指要害：「共同的问题就是 sim-to-real gap。」这些仿真器的预测能力受限于**参数调优**和**边界条件设置**的困难。「每一个用 simulator 做 robotics 的人应该都知道，你要把这些参数都调对，你要把这些 boundary condition 都设对，让这些 simulator 发挥到最好的状态，其实是相当难的一件事情。」

> **[重要]** **仿真的三难困境**
> 传统物理仿真面临三重张力：
>
> 1. **精度**：数值求解 PDE 的精度受网格分辨率、时间步长等限制
> 2. **真实性**：仿真器中的物理模型（如 Hooke 摩擦定律）只是对真实世界的近似
> 3. **效率**：高精度 + 高真实性的仿真在计算上是极其昂贵的
>
> 传统方法在这三个维度之间做 trade-off；而胡渊鸣认为，引入 neural network 可以帮助打破这个三角。

### 神经符号方法：Simulation + Neural Network

胡渊鸣提出的解决方案并非抛弃物理定律，而是**融合**：「用 1T flops 去预测一个 E flops 就能做好的事情，当然不 make sense。但是你也考虑到整个物理世界的复杂性，你也没有办法直接去用牛顿定律、欧拉-拉格朗日方程或者 Navier-Stokes 去 model——它们其实都是近似。那这个时候你可能需要很多 learnable 的系统去帮助你 capture 这些在真空环境下 work 的物理定律，在现实条件下到底是怎样的。」

关键原则：**「牛顿第二定律不需要被 transformer 重新学一遍。」** 已知的、廉价可解的物理定律应该保留为硬约束（hard constraint），而难以建模的边界条件和摩擦模型应由 neural network 来学习校正。

> **[知识]** **神经符号仿真的架构愿景**
> 一个可能的架构是：
>
> - **符号层（Symbolic Layer）**：保留牛顿力学、Navier-Stokes 方程等作为确定的物理先验，用高效数值方法求解
> - **神经层（Neural Layer）**：学习修正——比如 friction model、contact dynamics、material deformation 等难以直接从第一性原理推导的部分
> - **接口层（Interface Layer）**：决定何时信任符号解、何时调用神经修正——这本身也可以是一个可学习的门控机制
>
> 最终得到的系统「既不是一个 simulator，也不是一个 neural network」，而是一个嵌入了 neural network 的 **3D inductive bias**——它比纯神经网络更具物理一致性，比纯仿真器更能适应真实世界的复杂性。

### Synthetic Data 的边界

在讨论 synthetic data（合成数据）在机器人中的应用时，胡渊鸣表达了一个与主流观点不同的立场：

「Synthetic data 永远只能用来 **enhance** 一个 model，它不能 build a model from scratch from the bottom up。因为你要去 train 一个 foundation model 的话，用 synthetic data——所有 data 都是 synthetic data，或者大部分 data 是 synthetic data——一定不会 work。」

原因在于：synthetic data 天生「follow 某些 rules」，而这些 rules 是人造出来的——「这些 rules 有多少的 coverage 都成问题，现实世界可能有 1 亿种情况，你的 rules 只能 cover 1 万条。」更不用说这些 rules 本身可能与真实的物理定律存在偏差。

这与 Tesla / 李飞飞团队等推崇仿真路线的观点形成张力。胡渊鸣的立场是：**仿真不能替代真实数据，但可以作为真实数据的有力补充**。仿真提供的是结构化的物理先验（inductive bias），而不是训练信号的唯一来源。

### 本章小结
物理仿真与 AI 的融合不是「谁取代谁」的问题，而是如何让两种不同的知识表征方式互补。牛顿定律提供廉价而准确的「硬知识」，neural network 提供适应真实世界复杂性的「软知识」。合成数据的正确角色是 enhancer 而非 foundation。太极的技术积累——尤其是其 differentiable simulation 能力——恰恰位于这个融合的交汇点上。

## Sora 冲击波：学习 vs 模拟

### 「Sora 革了老 graphics 人的命吗？」

面对主持人「5 年前你的 99 行代码写冰雪奇缘，现在可能只需 20 个单词的 prompt——这样的 video model 是不是革了老 graphics 人的命？」的提问，胡渊鸣的回答坦诚而复杂：

**「有的场景是的吧。」**

他承认，对于某些生成任务（如创建一段飞行的龙喷火的视频），Sora 等 video diffusion model 的强大能力确实使传统 graphics pipeline 显得笨重。但「graphics 在做的很多事情，其实促成了 Sora 的诞生。」GPU 硬件、渲染管线、3D 表示——这些都是 graphics 领域的遗产，Sora 建立在它们的基础之上。

### Token 模型的根本局限

胡渊鸣从计算效率的角度揭示了当前 LLM/Video Model 范式的一个深层限制：

「不管这个 token 是多大的信息量，它一定是要花费两倍的 parameter count 除以 sparsity（如果用了 MoE）的 compute。现在在做 AGI 的尝试，就是把这个 scale 越搞越大越大，使得每一个 token 的 compute 和 evolve 到的权重越来越多。但是**很多的 token 它其实没有办法通过纯 scale 来解决**——即使你的 compute 已经很多了，有些 token 就是比其他 token 需要更多的 compute，这些 compute 不能发生在 LM 现在这个范式里面，而是要让 LM 去 **manipulate 一个工具**。」

这里的「工具」正是 simulation/graphics 所提供的结构化能力。例如：camera projection 在 traditional graphics 里面是 50 flops 的事情，「但是你用 AI 去做，你可能要加上多少个变量的参数？」

> **[重要]** **3D Inductive Bias 的价值**
> 胡渊鸣指出的核心问题是：纯 token-based model（如 Sora、Video Diffusion）缺乏对三维世界的**物理一致性**的显式建模。后果是：
>
> - 相机转 360 度回来，看到的是完全不一样的世界（缺乏 3D persistency）
> - 物体的遮挡关系、光影一致性难以保证
> - 无法进行精确的物理交互（如抓起一个物体并预测其运动）
>
> 而在 graphics pipeline 中，这些问题早已被精确建模。未来的方向应该是：让 neural network 学会**何时调用 graphics 原语**（如投影变换、光线追踪、碰撞检测），而非试图用神经网络重新学习这些已经解决了几十年的问题。

### Hybrid 的未来

胡渊鸣预见的未来是 hybrid 的：「以后肯定是 learning 为主，但是 simulator 会以某种形式——通过 inductive bias 也好、通过 synthetic data 也好——变成这个 learning 系统的组成部分。」他认为，graphics 为 video model 提供了**「物理世界的约束」**，这种约束不是限制，而是让模型不必从零开始学习每一个像素的运动规律。

他用太极 2.0 的愿景举例：LLM 领域还有很大的场景可以让太极去做——比如作为 LLM 操纵世界模型的底层计算引擎。这既是技术判断，也是商业判断：找到一个不会被纯 scale 淹没的独特价值点。

### 本章小结
Sora 对传统 graphics 的冲击是真实的，但不是终结性的。核心矛盾在于：token-based model 的计算效率在物理一致性问题上存在根本瓶颈。Graphics 提供的是「用 50 flops 解决 camera projection」这样的廉价精确计算能力，而 learning 提供的是处理视觉多样性和语义理解的能力——两者的结合才是通向真正物理世界理解的路径。

## 创业生死转型：从太极到 Meshy

### 「bias for action」：先开枪，后瞄准

胡渊鸣的创业起点带有强烈的行动主义色彩：「我非常 bias for action，一个事情我想做我立马就去做，晚上不睡觉我也去做。」他甚至用 Elon Musk 的比喻来形容自己的思维方式——「他还没火箭发动机还没造好呢，他就想着火车在火星上面我们吃什么？我觉得这个是我的一个特点。」很多人觉得这是「八字还没一撇」，但他认为**以终为始的思考**才是驱动创业的核心动力。

2021 年博士毕业后，他没有选择去大厂，而是直接创办了太极图形。「我受不了有人告诉我该做什么。」他对控制权的执着源于自己的性格底色，也与 MIT 期间的独狼经历一脉相承。

### 前 18 个月的黑暗摸索

然而，「先开枪」的代价是前 18--24 个月的迷茫。太极图形最初的方向是继续做太极编程语言本身——支持更多硬件（AMD GPU）、甚至开发一个 JavaScript backend 让太极能在网页中运行。但在商业化上始终没有起色。

「试了一些商业化的方式，最后发现要做商业化只能做外包。」这意味着一家由 MIT 博士、清华姚班毕业生创办的公司，要靠帮别人开发 simulation 项目来维生——收钱、开发、交付，周而复始。

> **[注意]** **开源商业化的经典困境**
> 太极遇到的困境是一个普遍问题：开源项目可以建立巨大的社区影响力和技术声望（太极的 GitHub star 数量至今仍超过 NVIDIA Warp、Taichi、Triton 等竞品），但将社区影响力转化为商业收入是极其困难的。胡渊鸣总结道：「开源是不挣钱的，但是你有一个好的开源项目，它能够帮助你吸引到最好的人才。」

### 「烟花秀」危机与果断转型

胡渊鸣在这段经历中提炼出一个关键原则：**「你做的技术或者产品必须能够商业化。如果一个技术没有办法商业化的话，你就没有办法用商业化得到的 revenue 去养活未来的技术的研究——这个技术本身就会变成一场烟花秀，放完烟花秀就没有了。」**

他将理想的商业模式描述为一个**飞轮**（flywheel）：

[图：TikZ 可视化 — 见 PDF]

2023 年，他做出创业以来最艰难的决定：**从太极编程语言正式转型到 3D 生成式 AI 平台 Meshy**。在这个过程中，「有一些阻力，有一些人觉得这个事没兴趣，就离开了公司。但是坚持做的人，现在都觉得这是一个非常正确的决定。」

### 为什么是 Meshy

胡渊鸣选择 3D 生成（text-to-3D）这个方向，并非因为它最容易，而是因为它与团队能力的匹配度最高：「我去做 LLM 或者去做 2D diffusion、text-to-image 这些东西或者是 text-to-video，我不见得做过其他人。我自己的能力局限性就导致其实 Meshy 在当时就是我最好的一个方向。这也是整个团队能力的优势所在。」

这里体现了一个创业者最稀缺的素质：**清晰认知自己的能力边界**。他没有去追 LLM 或 video generation 的热点，而是选择了自己团队拥有独特优势的 3D 生成赛道。

他还有一个重要的认知：创业初期要「build 一个这个世界需要的东西」而非「build 自己想要的东西」。他用一个具体的发现来说明这一点：「之前的产品虽然做失败了，但是我们聊了大量的用户，他们都说我不会为你这个软件付费的——但是你如果能把这个里面的模型都卖给我，我会为他付费。」

> **[重要]** **转型决策的核心框架**
> 胡渊鸣的转型决策可用三个维度概括：
>
> 1. **市场需求**：用户愿意为 3D asset 付费（而非为 simulation 工具付费）
> 2. **团队能力**：graphics + compiler 背景是 3D 生成的独特优势，而非劣势
> 3. **竞争格局**：2D/LLM 已有巨头林立，3D 生成尚是蓝海
>
> 三者交叉的 sweet spot 就是 Meshy 的定位。

### 本章小结
从太极到 Meshy 的转型是胡渊鸣创业生涯中最关键的一次「自我杀死」。它验证了他自己的哲学——「每 3--6 个月就要把自己开除一次」。转型的痛苦是真实的（有人离开），但剩下来的人证明了方向是正确的。核心教训：开源创造影响力但难变现，商业化必须找到用户愿意付费的点，而技术团队的独特能力决定了你该选择哪个赛道。

## Meshy AI：产品、市场与增长

### 「恐怖游戏专用」到市场第一

Meshy 的起步并不光鲜。胡渊鸣坦言：「最早的时候只能用来做恐怖游戏。为什么只能做恐怖游戏呢？因为我们生成的所有的模型都有四张脸。」

这个听起来像段子的开局揭示了 AI 3D 生成的早期质量有多差——模型会有 4 张脸（可能是多视角生成时的 ghosting artifact）。但随着技术迭代，「现在能解决的问题已经比原来大了很多很多」。

Meshy 如今在美国发达市场的**市场占有率达到 55%**，每月保持**20% 的增长**，年营收增长超十倍。主要商业模式是**订阅制**，辅以 to B 的 API 服务和 studio account。

### 意想不到的最大客户

一个反直觉的发现：Meshy 的最大客户不是游戏公司，而是**Meta**。「大厂里面可能有些 API 的需求，那这个其实比游戏公司要更大一些。」这与许多人对 3D 生成工具的直觉相反——人们自然以为游戏行业才是最大买家。

不过，游戏行业确实是 Meshy 的重要市场。3A 游戏制作成本的 50% 都在 3D asset 上，这是一个巨大的降本增效空间。但整个游戏行业近年在收缩：「大家都忙着降本增效，忙着涨价，忙着把以前老游戏做个重制版再放出来。」

胡渊鸣分析了游戏行业萎缩的几个原因：TikTok 等短视频平台抢走了用户时间；吃鸡（PUBG）之后已经很长时间没有新的爆款玩法出现；免费游戏 + 内购的商业模式也遇到了瓶颈。

### AI 3D 生成的成熟度

2023 年，胡渊鸣在访谈中说 Meshy 只解决了 AI 3D 生成技术问题的 10%，期望 2025 年解决 90%。当主持人追问现在的进度时，他的回答坦诚而有趣：

**「也许 50%。因为你越解决越发现，上限并不是 100%。所以我觉得现在可能解决了当时想象的 90% 吧，但是后来发现哦，上限是 200%。」**

这是一个经典的技术探索曲线：随着认知加深，不是问题变简单了，而是你看到了更多问题。但他认为剩下的 50% 在可预见的未来是 solvable 的。

「我在这过程中学到最大的一个事情就是——**不要低估技术前进的速度**。」

> **[知识]** **Meshy 的产品哲学**
> 胡渊鸣反复强调的一个原则是：用户付费意愿是检验产品价值的唯一标准。「之前的产品虽然做失败了，但是我们聊了大量的用户，他们都说我不会为你这个软件付费的——但是你如果能把这个里面的模型都卖给我，我会为他付费。」
>
> 这一洞察直接塑造了 Meshy 的产品形态：从卖「仿真开发工具」变为卖「3D 资产」。前者是 developer tool，后者是 creative asset——后者离用户的 ROI 更近，也更容易产生 repeat purchase。

### 书的文化 vs 赢的文化

在产品之外，胡渊鸣花了大量时间讨论企业文化。他提出了一个鲜明的二分法：**「书的文化」与「赢的文化」**。

「书的文化」的特征是：内耗、甩锅、不面对事实、粉饰太平、不能批评 CEO、追求和谐。他特别强调：「出了问题，我不去复盘、不去反思，或者说团队不能批评 CEO、不能再公开指出其他人的问题——这些我觉得都是会输的文化。」

而「赢的文化」的核心是**尊重事实**：「我明明做的事情是这样，我一定要把它粉饰成是另外一样。」不面对事实的公司注定失败。

胡渊鸣分享了一个近期的个人经历：「我前天刚被人骂了一顿——一个同学把我和另外一个同学，三个人把我拉到一边走廊里边说，我有事情没有做好。」他的反应不是防御，而是「心头一紧，然后听他说完，再想自己是不是真的没做好，后来发现确实有没做好的地方。」

> **[重要]** **创新驱动的组织需要什么**
> 胡渊鸣区分了两种组织形态：
>
> - **制造业公司**：CEO 可以更强势，指令可以自上而下
> - **创新驱动公司**：「你要接受 CEO 被挑战、被批评。因为如果你不接受这些，你就不知道你做的事情到底对不对。」
>
> Meshy 显然属于后者。招到愿意当面批评 CEO 的人，在胡渊鸣看来「是非常幸运的——他一定是对这个公司有自己很强的 ownership，然后希望这事能搞成，他把自己当 CEO 一样，才会愿意跟你讲这些。」

### 本章小结
Meshy 的成长故事是一个经典的「找准 PMF（product-market fit）」案例。从无法商业化的太极语言到 55% 市场占有率的 Meshy，关键转折点在于：发现了用户愿意付费的 point（3D asset 而非 simulation tool），并持续快速迭代——从「四张脸的恐怖游戏专用」迭代到能在竞争激烈的美国市场占据主导地位。但胡渊鸣清楚，AI 3D 生成距离「让 studio 满意」还有很长的路要走——「往往还是要去修复的，再用 ZBrush 再去修一修。」

## CEO 的修行：领导力、决策与文化

### 时间分配：招募是 CEO 的第一要务

胡渊鸣对 CEO 工作的自我定义是三个板块：

- **招募（40%）**：找到最好的人，让他们做最好的事
- **思考未来（30%）**：定义公司的 roadmap 和 strategy
- **紧密参与业务（30%）**：hands-on 地了解公司在做什么

招募的重要性远超一般人想象。胡渊鸣说：「哪怕是公司的战略方向、我们要做什么 product，都比不上招更好的人进来。」

那么什么样的人才是他想要的？他的答案高度依赖直觉，但有共性：**清晰度（clarity）**。「说很多但只有 10% 的内容是对方能够接收的——我在很多新管理者身上能看到这样的问题。我自己刚成为管理者的时候也会有这样的问题。直到我后来发现——说不重要，听最重要。」

### Tough Decision：CEO 的孤独

胡渊鸣对「tough decision」的定义直白得几乎残忍：

- 「砍掉一个大家都很喜欢的项目」
- 「请走一个对团队没有贡献的同事——但是大家都很喜欢他，他是个好人」
- 「你做了 A 喜欢的，B 就要骂你；你做了 B 喜欢的，A 就要骂你；都做，公司就没了」

做 CEO 意味着你要为整个公司的利益负责，而不是让自己被所有人都喜欢。**「你坐在 CEO 的位置上，你就要为整个公司的利益去负责，而不是让自己被所有人都喜欢。」**

他把这个过程比作打 Boss：「人生中其实每时每刻都有阶段的变化——为什么每 3--6 个月都要成为一个全新的自己？你如果不去每 3--6 个月变成一个新的自己的话，你就永远卡在了上一个阶段。如果你的公司每个月增长 20% 的话，三四个月够你营收翻倍了——那你还还在用以前那种方式去处理现在的问题吗？」

> **[注意]** **阶段一的成功 = 阶段二的陷阱**
> 胡渊鸣识别了一个危险的模式：阶段一（打工/学术界）越成功的人，往往阶段二（创业/CEO）越容易失败。原因在于技能完全不兼容：
>
> - **阶段一**：服从指挥、卷效率、磨练具体技能。平台提供确定性，你只需要负责执行。
> - **阶段二**：没人再指挥你了。效率无从谈起（连效率的 metric 都没有定义）。你需要自己 figure out「什么是正确的事」。
>
> 「就跟打 boss 一样，一个血条打完了，现在变形了，打第二个血条。你还用第一个血条的那个方式去打，很快就挂了。」

### Clarity：管理者的核心能力

胡渊鸣反复回到「clarity」这个词。他认为 clarity 有三种表现：

1. **向上 clarity**：能把复杂问题清晰地向上汇报，让决策者做出正确判断
2. **向下 clarity**：能把战略方向清晰传达给团队，让每个人的行动对齐
3. **对外 clarity**：能确保表达的所有观点都清晰地传递给了听众

但他最强调的 clarity 维度是**听的能力**：「说不重要，听最重要。」听到真正的用户需求、听到团队的真实反馈、听到市场在发生什么变化——这些比说漂亮话重要得多。

### 本章小结
胡渊鸣的 CEO 哲学围绕三个核心展开：招募最优秀的人（因为他们决定了公司的一切上限），做 tough decision（因为回避困难只会让问题变大），保持 clarity（因为混乱是组织最大的效率杀手）。他将这些能力视为可习得的——他自己就是从「会说很多但 10% 被接收」的新管理者变成了一个强调倾听的领导者。最危险的陷阱是：用旧阶段的成功方法去应对新阶段的挑战。

## 人生哲学：勇气、智慧与持续进化

### 「每 3--6 个月杀死过去的自己」

这是本期播客最核心的哲学命题，也是胡渊鸣反复回响的主题。

「我在做公司的这个过程中，每 3--6 个月就要把自己开除一次。」这不是夸张。如果一个公司每月增长 20%，3--4 个月就翻倍——去年有效的工作方式，今年可能就是瓶颈。如果 CEO 不主动「杀死」过去的自己，增长的飞轮自己会停下来。

[图：TikZ 可视化 — 见 PDF]

### AI for Fun：AGI 时代的 Vision

胡渊鸣对未来的愿景出人意料地「温和」——他强调的是**「AI for fun」**。

「我觉得为什么 AI for fun 这个事情对我来说很有吸引力？因为我觉得 AGI 来了以后，所有人都要失去工作。然后一个更好的世界就是大家还能开开心心地活着，过得开心就好。」

这个看似轻松的说法背后是一个严肃的赌注：随着 AI 接管越来越多的生产性工作，人类的**情感价值、创造乐趣和陪伴需求**将成为新的经济引擎。Meshy 的终极愿景不是让 3D artist 失业，而是让**每个人都能创造 3D 内容**——「你可以让你父母来用」。

他举了一个个人化的例子：他上传了自己和女友的合照，让 Meshy 生成了一个 3D asset。女友的第一反应是「太丑了」，但「如果你可以把它做的很好看的话，她会愿意 3D 打印它。」

### Visioning：不是一次性的 Vision

胡渊鸣做了一个微妙的词义区分：不是「have a vision」，而是**「visioning」**——一个持续进行的过程。

「整个市场会不停的变化，你的认知会不停的变化。所以你每时每刻都要想着未来到底是会是什么样的——5 年以后的世界，10 年以后的世界会是什么样的。」

这种动态的 visioning 与他「每 3--6 个月杀死自己」的哲学一脉相承。Vision 不是一个静态的目标，而是一个需要持续校准的 compass。

### 勇气与智慧：留给自己和世界的话

播客尾声，主持人让胡渊鸣留一句话作为 2025 年的时间胶囊。他的回答简洁有力：

**「希望所有看到这个 podcast 的人，都能具备勇气。」**

这与他反复提及的另一个关键词「智慧」形成对仗。在他那篇广为流传的知乎文章「当 CEO 重读 PhD：论智慧与勇气」中，他花了大量篇幅探讨这两个品质。勇气体现在「敢做」——敢在博士期间造编译器、敢在没人看好时创业、敢在 18 个月零商业化时果断转型。智慧体现在「知道该做什么」——知道自己 enjoy 什么、知道团队能力的边界、知道何时该「杀死」上一个自己。

> **[重要]** **胡渊鸣的「勇气—智慧」框架**
> 本期播客可以被理解为一份关于**如何在不确定中做决策**的案例研究：
>
> - **勇气（Courage）**：驱动「开始」——辞掉确定性、启动新方向、做出 tough decision
> - **智慧（Wisdom）**：驱动「选择」——识别自己的 enjoy 点、感知能力边界、判断市场信号
> - **两者的循环**：勇气让你进入新阶段，智慧帮你在这个阶段存活并找到迭代方向；然后新的阶段需要新的勇气去打破旧模式
>
> 缺少勇气，你会停在舒适区；缺少智慧，你会死在转型路上。两者缺一不可。

### 本章小结
胡渊鸣的人生哲学可以从三个维度理解：时间维度上，他主张「每 3--6 个月杀死过去的自己」；空间维度上，他追求「AI for fun」——让技术服务于人的快乐而非仅仅是效率；决策维度上，他强调勇气与智慧的双轮驱动。这不仅是一个 CEO 的自我修养，更是一个在高速变化时代中的生存策略。

## 总结与延伸

### 胡渊鸣的核心主张

这场三小时对话的核心信息可以浓缩为以下几个命题：

1. **自我认知是唯一不可外包的能力**：「你知道你喜欢什么」比「你有多聪明」重要一百倍。胡渊鸣从小学就知道自己 enjoy 编程，这个 clarity 贯穿了他所有重大选择。
2. **技术必须商业化才能持续**：「不商业化的技术是一场烟花秀——放完就没了。」从太极开源到 Meshy 转型，是整个对话中最关键的一次范式跃迁。
3. **物理世界不可被纯 learning 替代**：牛顿定律不应该被 transformer 重学一遍。Simulation 和 neural network 的关系是互补而非替代——前者提供廉价的物理一致性，后者填补真实世界的复杂性 gap。
4. **每 3--6 个月杀死过去的自己**：增长本身会制造新的瓶颈。旧阶段的成功方法是新阶段的毒药。持续进化的前提是持续否定。
5. **组织的胜负取决于是否尊重事实**：「输的文化」的核心特征是不面对事实。而「赢的文化」让最 junior 的人也有勇气把 CEO 拉到走廊里说「你没做好」。

### 关键洞见与开放问题

**已验证的洞见**：

- 开源的商业价值不在于直接变现，而在于吸引人才和建立技术品牌——太极的 star 数超过竞品，但无法支撑一家公司
- 3D 生成的市场需求真实存在：55% 美国市场份额、月增长率 20%、Meta 作为最大客户——这些数据说明 PMF 已经成立
- CEO 的「听的能力」比「说的能力」更重要：信息获取的质量决定了决策的质量

**开放问题**：

- 胡渊鸣认为 hybrid simulator（物理仿真 + 神经网络）是未来方向，但具体的架构设计、训练方法和验证标准尚未成熟。太极 2.0 能否在这个方向上破局？
- AI 3D 生成距离「让 studio 满意」还有质变要走。从 50% 到 200% 上限的 gap 中，哪些是 scaling 问题（更多数据/更大模型就能解决），哪些是方法问题（需要全新的架构）？
- 「每 3--6 个月杀死自己」的哲学在个人层面有效，但 scaling 到组织层面（几十人、几百人）会遇到什么阻力？「书的文化」天然抗拒被杀死——如何设计机制让组织也能持续自我革新？
